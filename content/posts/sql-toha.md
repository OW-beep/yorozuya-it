## SQLとは「データベースに指示を出すための言葉」

SQL(エスキューエル)とは「Structured Query Language」の略で、**データベース(大量のデータが整理して保管されている場所)に対して、データの取得・追加・変更・削除を指示するための言語**です。「エスキューエル」のほか、「シークェル」という読み方をする人もいますが、どちらも同じSQLを指しています。プログラミング言語というよりは、データベースと会話するための専用の言葉、というイメージが近いです。

## データベースとは何か

まずデータベースについて簡単に触れておきます。データベースとは、表計算ソフトのような「行」と「列」からなる表を、大量かつ効率的に管理できるようにした仕組みのことです。ネットショッピングサイトの商品一覧や、会員情報の管理など、私たちが普段目にする多くのサービスの裏側で使われています。

## 実際のSQL文を見てみる

言葉だけの説明よりも、実際の文を見た方がイメージが掴みやすいかもしれません。例えば「会員」という表から、30歳以上の会員の名前とメールアドレスだけを取り出したい場合、SQLでは次のように書きます。

```sql
SELECT 名前, メールアドレス
FROM 会員
WHERE 年齢 >= 30;
```

「SELECT(選ぶ)」で欲しい項目を、「FROM(〜から)」でどの表を対象にするかを、「WHERE(〜という条件で)」で絞り込み条件を、それぞれ英単語に近い言葉で指定しているのがわかります。この読みやすさこそが、SQLが「特別な才能がなくても習得できる」と言われる理由の1つです。

## SQLの基本4命令

SQLで行える操作は、大きく分けて次の4つに集約されます。それぞれ英語の頭文字をとって「CRUD(クラッド)」と呼ばれることもあります。

| 命令 | 意味 | 操作の例 |
|---|---|---|
| SELECT | 取得する | 条件に合うデータを一覧表示する |
| INSERT | 追加する | 新しい会員情報を1件登録する |
| UPDATE | 変更する | 特定の会員の住所情報だけを書き換える |
| DELETE | 削除する | 退会した会員のデータを消す |

この4つの命令の組み合わせだけで、データベースに対するほとんどの操作をまかなうことができます。

## SQLの文がとても英語に近い理由

SQLの大きな特徴は、他のプログラミング言語と比べて、比較的英語の文章に近い書き方をする点です。例えば「30歳以上の会員を選んで」という指示は、実際のSQLでも「SELECT(選ぶ)」「WHERE(〜という条件で)」といった、英単語に近い言葉で組み立てられます。この読みやすさから、プログラミング未経験者でも比較的習得しやすい言語だと言われています。

## SQLが使われる「リレーショナルデータベース」について

SQLは、正確には「リレーショナルデータベース」と呼ばれる、表と表を関連づけて管理するタイプのデータベースを操作するための言語です。一方、近年は「NoSQL」と呼ばれる、表形式にとらわれない、より柔軟な構造でデータを管理する仕組みも広まっています。NoSQLではSQL以外の操作方法が使われることが一般的で、扱うデータの性質によって、どちらの仕組みが適しているかが使い分けられています。

## どんな職種で使われているか

エンジニアだけでなく、マーケティング担当者やデータ分析担当者が、必要なデータを自分で取り出すためにSQLを学ぶケースも増えています。「大量のデータの中から、必要な情報だけを取り出す」というニーズは、多くの職種に共通しているためです。

運営者自身も学習中に同じ疑問を持ったことがあり、その経験も踏まえてまとめています。

## まとめ

SQLは、膨大なデータの中から必要な情報だけを取り出すための「データベースとの会話の仕方」です。SELECT・INSERT・UPDATE・DELETEという4つの基本命令さえ押さえれば、基本的な操作の大部分をカバーできます。比較的とっつきやすい言語なので、興味があれば触れてみる価値は十分にあります。
